Den ComfyUI Manager installieren

Der Manager ist das wichtigste Add-on für ComfyUI: Er installiert Custom Nodes und Modelle per Klick – inklusive aller Abhängigkeiten.

Basics Setup Custom Nodes

Wofür der Manager?

ComfyUI im Auslieferungszustand kennt nur seine Standard-Nodes. Die meisten interessanten Workflows nutzen aber Custom Nodes – Erweiterungen von der Community. Der ComfyUI Manager installiert, aktualisiert und verwaltet diese Erweiterungen über eine Oberfläche, statt dass du manuell mit Git und Abhängigkeiten hantierst.

Der Vorteil: Lädst du einen fremden Workflow, der unbekannte Nodes enthält, findet der Manager per Install Missing Custom Nodes automatisch die passenden Erweiterungen. Genau deshalb installieren wir ihn früh.

Den Manager in der Portable-Version aktivieren

In den aktuellen Portable-Versionen ist der Manager bereits Teil von ComfyUI – er ist nur standardmäßig noch nicht eingeschaltet. So aktivierst du ihn:

  1. Die Konsole öffnen

    Die Konsole ist das Fenster, in dem Befehle per Text laufen. Öffne im Windows-Explorer deinen Ordner ComfyUI_windows_portable, klicke oben in die Adresszeile, tippe cmd und drücke Enter. Die Konsole öffnet sich direkt in diesem Ordner – nichts weiter einzustellen.

  2. Einen Befehl hineinkopieren

    Kopiere die folgende Zeile komplett (Strg+C), füge sie in der Konsole mit Rechtsklick ein und drücke Enter. Damit lädt ComfyUI einmalig die Bausteine nach, die der Manager braucht:

    .\python_embeded\python.exe -m pip install -r ComfyUI\manager_requirements.txt

    Kurz warten, bis das Fenster fertig durchgelaufen ist (es zeigt am Ende wieder eine Eingabezeile an). Dann kannst du es schließen.

    Nur falls deine Version den Manager noch nicht eingebaut hat: Lade dir aus dem offiziellen Repo die Datei install-manager-for-portable-version.bat in deinen ComfyUI_windows_portable-Ordner und starte sie per Doppelklick. Das erledigt dasselbe ganz ohne Befehle.

  3. Den Manager beim Start mit einschalten

    ComfyUI startest du normalerweise über die Datei run_nvidia_gpu.bat. Damit der Manager mitgeladen wird, musst du dieser Datei ein kleines Stück Text anhängen. Mach einen Rechtsklick darauf → Bearbeiten (öffnet sich im Editor). Ganz am Ende der Zeile, die mit .\python_embeded beginnt, schreibst du – nach einem Leerzeichen – --enable-manager --enable-manager-legacy-ui dahinter. Die Zeile sieht dann so aus:

    .\python_embeded\python.exe -s ComfyUI\main.py --windows-standalone-build --enable-manager --enable-manager-legacy-ui

    Speichern (Strg+S), Editor schließen, und ComfyUI wie gewohnt per Doppelklick auf die .bat starten. Nach dem Start erscheint oben der Manager-Button. Fertig – das machst du nur dieses eine Mal.

Warum über die Adresszeile? So öffnet sich die Konsole direkt im richtigen Ordner und der Befehl findet die passenden Dateien von allein. Alternativ geht es über das Startmenü (Eingabeaufforderung).

Warum zwei Schalter? ComfyUI bringt seit Ende 2025 eine neue Manager-Oberfläche mit. Beide Varianten stecken in derselben Installation – nur die Ansicht unterscheidet sich. Wir verwenden in diesem Tutorial die klassische Oberfläche (--enable-manager-legacy-ui), weil sie alle Funktionen an einem Ort bündelt – inklusive Model Manager und Update All, die in der neuen Ansicht (noch) fehlen. Wenn du lieber die neue Oberfläche ausprobieren möchtest, lass den zweiten Schalter einfach weg: Du findest den Manager dann über das Plugin-Symbol oder über Menü → Manage Extensions.

Der Manager im Überblick

Ein Klick auf den Button öffnet das Manager-Fenster. Die wichtigsten Funktionen:

Install Custom Nodes – durchsuche und installiere Erweiterungen. Install Missing Custom Nodes – ergänzt fehlende Nodes eines geladenen Workflows. Model Manager – lädt Modelle direkt herunter. Update All – hält alles aktuell. Nach jeder Installation einfach ComfyUI neu starten.

Abhängigkeiten? Übernimmt der Manager. Installierst du eine Erweiterung, zieht er die benötigten Pakete automatisch mit – du musst nur neu starten.
